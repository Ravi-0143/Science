# Atlas Design Language

**Version:** 1.0

---

## Design Language

**Atlas should feel:**
Elegant · Calm · Precise · Immersive · Curious · Timeless · Scientific · Premium · Minimal · Spatial

**Atlas should never feel:**
Flashy · Chaotic · Over-designed · Game-like · Neon-heavy · Cluttered

---

## Components

Experiences are composed from reusable components. Examples include:

Hero, Explorer, Timeline, Simulation, Comparison, Story, Quiz, Challenge, Gallery, Lab, 3D Viewer, Diagram, and Knowledge Card.

The platform should provide these building blocks. Experiences assemble them.

---

## Motion

Motion should communicate. Camera movement should create presence. Depth should be preferred over opacity. Transitions should feel physical.

Avoid abrupt changes. Avoid unnecessary effects. Smoothness is more important than speed.

---

## Visual Identity

- Whitespace is a feature.
- Typography should communicate hierarchy.
- Color should communicate meaning.
- Lighting should guide attention.
- Depth should create immersion.
- Every screen should feel intentional.

---

## The Craft Standard — What "Apple" Means Here

Feeling is delivered *through* craft. This is the layer sloppy execution kills fastest, and the one most specs leave unwritten.

- **Typography leads.** Large, confident, restrained. One statement on screen at a time — never text competing with an interaction for attention.
- **Whitespace is a feature.** Let every idea breathe before the next one arrives. Empty space isn't unfinished space.
- **One idea per scene.** If it needs two ideas, it needs two scenes.
- **Motion is physical, not decorative.** Easing should feel like something with real weight moved. Nothing snaps, nothing is gratuitous — motion clarifies relationships, it doesn't perform.
- **Color is intentional and sparse.** A calm, neutral canvas, one accent that actually means something.
- **Every interaction has a crafted response.** Hover, press, drag, release — each considered, none left at browser default.
- **Complexity is hidden.** The learner never sees a setting, a label, or a technical term they didn't need in order to discover something.
- **Nothing ships half-built.** If a scene isn't ready, it doesn't load in front of the learner.
- **Consistency is non-negotiable.** Every experience, whatever the topic, should feel like the same hand made it.

Quick test: if a screen would look at home on apple.com, keep going. If it would look at home in a textbook PDF, start over.
